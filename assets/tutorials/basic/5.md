## Generate data with liquibase

We are now going to add data to our database when we start it up.

### Use file includes

[Liquibase Includes](http://www.liquibase.org/documentation/include.html) allows you to break down your database changes in smaller, manageable pieces. 

Add the following to db.changelog-master.yaml
```yaml
databaseChangeLog:
  - include:
      file: "db/changelog/v1.yaml"
```

### Create table for administrators

1. Create a file called v1.yaml and add the following to it
```yaml
databaseChangeLog:
  - changeSet:
      id: 1
      author: thor
      changes:
        - createTable:
            tableName: administrators
            columns:
              - column:
                  name: id
                  type: int
                  autoIncrement: true
                  constraints:
                    primaryKey: true
                    nullable: false
              - column:
                  name: username
                  type: varchar(255)
                  constraints:
                    nullable: false
              - column:
                  name: password
                  type: varchar(255)
                  constraints:
                    nullable: true
              - column:
                  name: email
                  type: varchar(255)
                  constraints:
                    nullable: false
              - column:
                  name: country
                  type: varchar(255)
                  constraints:
                    nullable: true
```

2. Insert data into the administrators table

```yaml
  - changeSet:
      id: 2
      author: thor
      changes:
        - insert:
            tableName: administrators
            columns:
              - column:
                  name: username
                  value: thor
              - column:
                  name: password
                  value: Welcome#1
              - column:
                  name: email
                  value: thor.odinson@medrec.com
              - column:
                  name: country
                  value: midgard

        - insert:
            tableName: administrators
            columns:
              - column:
                  name: username
                  value: ironman
              - column:
                  name: password
                  value: Welcome#1
              - column:
                  name: email
                  value: tony.stark@medrec.com          
              - column:
                  name: country
                  value: USA       
```
3. Run the application:
```bash
mvn -Pdev spring-boot:run
```
4. Access the H2 web console: [http://localhost:8080/h2-console](http://localhost:8080/h2-console)
5. Login and click on the ADMINISTRATORS table. This should automatically add the query:
```SQL
SELECT * FROM ADMINISTRATORS 
```
6. Execute the query. You should see the 2 users created above being returned.

7. Repeat steps 1-6 for Patients, Physicians, Prescriptions.
